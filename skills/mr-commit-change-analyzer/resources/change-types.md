# Change Type Classification

Use this standard classification when analyzing code changes:

| Type | Description |
|------|-------------|
| **Feature Addition** | New functionality, endpoints, UI components, or business logic |
| **Bug Fix** | Correction of incorrect behavior, crash fixes, or error handling |
| **Refactoring** | Code structure changes without altering functionality (renaming, restructuring, cleanup) |
| **Performance** | Changes aimed at improving system performance, reducing latency, or optimizing resource usage |
| **Documentation** | Updates to README, comments, docstrings, or external documentation |
| **Style/Formatting** | Code style changes, whitespace adjustments, formatting that doesn't affect logic |
| **Test Changes** | Addition, modification, or removal of test cases |
| **Dependency Update** | Changes to package.json, requirements.txt, or other dependency files |
| **Configuration** | Updates to config files, environment variables, or deployment settings |
| **Breaking Change** | Modification that breaks backward compatibility with existing APIs or interfaces |
