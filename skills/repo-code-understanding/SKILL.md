# Repository Code Understanding

## Description
This skill provides comprehensive code understanding for large-scale code repositories. It supports splitting analysis into manageable subtasks, generates structured documentation organized by module, and allows incremental updates based on new MR/commit changes. It can handle monorepos and large codebases efficiently without requiring full reanalysis on every update.

## Usage Scenario
Trigger this skill when:
- The user asks to "understand this repository" or "generate code documentation"
- Onboarding to a new codebase and need to understand the overall architecture
- Need to maintain up-to-date code documentation as the project evolves
- Analyze large codebases that cannot be processed in a single pass
- Incrementally update documentation after new MRs/commits are merged

## Instructions

### Full Analysis Flow
1. **Assess**: Scan structure, identify modules/layers, estimate complexity, and split into subtasks.
2. **Process**: Analyze each module's responsibility, API, dependencies, data flow, and patterns.
3. **Generate**: Combine results into `code-docs/` (README, Architecture, Modules). Use templates below.

### Incremental Flow
1. **Detect**: Identify modules affected by MR/commit changes.
2. **Reanalyze**: Update docs for affected modules. Update architecture if significant.
3. **Update**: Merge changes into main docs and update changelog.

## Analysis Strategies

- **Prioritize**: Entry Points -> Core -> Features -> Infra -> Config.
- **Split**: Break large modules (>50 files) by subdomain.
- **Deps First**: Analyze dependencies before dependents. Map circular deps.
- **Batching**: Group related modules.
- **Abstraction**: Maintain High (summary), Medium (interaction), and Low (implementation) levels.

## Module Classification

- **Core**: Business Domain, Shared Utils, Foundation UI, Infra Abstractions.
- **Feature**: User-facing, Admin, Integration, Experimental.
- **Infra**: API Layer, Data Access, Auth, Background Jobs, Deploy/Config.
- **Docs**: Public API, Internal Tools, Tests.
- **Complexity**: Low (<10 files), Medium (10-50), High (>50 or critical path).

## Incremental Update Rules

- **Impact**: Trivial (skip), Minor (update section), Significant (reanalyze module), Architectural (update system).
- **Triggers**: >20% file change, API change, logic change, dependency change.
- **Cross-Module**: Update dependents if API changes.
- **Conflict**: Flag ambiguous changes for human review.
